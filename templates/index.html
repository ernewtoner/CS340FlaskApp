Project Group 8: Film Tracker

Eric Newton & Katie Young

Project Outline

We will be creating a database representing a film catalog. This catalog can be used to track films, their genres, awards they received, and directors and starring actors. The database stores various attributes for each film, genre, award, director and actor that we believe will be useful in providing context for the various relationships listed below. With multiple many-to-many relationships, plenty of data to work with and several attribute-heavy entities, the complexity of this project is conducive to learning in the context of this course.

Database Entities & Attributes

  • Films:
    • ID: A unique identifier that is automatically assigned to each film as it is recorded in the database. This integer is automatically incremented and serves as the primary key.
    • Title: Title of the film, a string of maximum 100 characters. It cannot be blank and there is no default.
    • Language: Language of the film, a string of maximum 50 characters. It cannot be blank but the default language is “English”.
    • Year: Year the film was released, an integer.
    • Runtime: Length of the film in minutes, an integer.
  • Actors:
    • ID: A unique identifier that is automatically assigned to each actor as it is recorded in the database. This integer is automatically incremented and serves as the primary key.
    • First Name: First name of the actor which is a string of maximum 50 characters. It cannot be blank and there is no default.
    • Last Name: Last name of the actor which is a string of maximum 50 characters. It cannot be blank and there is no default.
    • Year Born: Actor’s birth year which is an integer. It cannot be blank and there is no default.
    • Year Died: Actor’s death year (if applicable) which is an integer. The default is NULL.
  • Directors:
    • ID: A unique identifier that is automatically assigned to each director as it is recorded in the database. This integer is automatically incremented and serves as the primary key.
    • First Name: First name of the director which is a string of maximum 50 characters. It cannot be blank and there is no default.
    • Last Name: Last name of the director which is a string of maximum 50 characters. It cannot be blank and there is no default.
    • Year Born: Director’s birth year which is an integer. It cannot be blank and there is no default.
    • Year Died: Director’s death year (if applicable) which is an integer. The default is NULL.
  • Genres:
    • ID: A unique identifier that is automatically assigned to each genre as it is recorded in the database. This integer is automatically incremented and serves as the primary key.
    • Name: A string of maximum 50 characters that names the genre. It cannot be blank and there is no default.
    • Description: A string of maximum 1000 characters that describes the genre. NULL if no description.
  • Awards:
    • ID: A unique identifier that is automatically assigned to each award as it is recorded in the database. This integer is automatically incremented and serves as the primary key.
    • Title: Title of the award, a string of maximum 100 characters. It cannot be blank and there is no default.
    • Year: Year the award was received, an integer. NULL if not awarded.

Relationship Breakdown

  • Film to Actor (many-to-many): a single actor can play in many films and many actors can star in a single film. Many actors can also star in many films.
  • Film to Director (many-to-many): a single director can direct many films and many directors can direct a single film. Many directors can also direct many films.
  • Film to Genre (many-to-many): a film is assigned only one genre and many films can be categorized under any given genre.
  • Film to Award (many-to-many): a single film can receive many awards and many films can receive the same or many different awards.
